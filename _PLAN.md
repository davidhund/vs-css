# Plan: `@davidhund/vs-css` ŌĆö Modern CSS Architecture Boilerplate

## Context

A greenfield CSS architecture template to bootstrap new web projects. Published as an npm package (`@davidhund/vs-css`) and usable as a template via `giget`. The deliverable is a `./css/**` folder structure with everything needed for a strong, modern CSS foundation ŌĆö focused on architecture and structure, not visual style. Wide open-source audience.

**Not in scope:** component-level CSS, a UI library, or any JS framework coupling.

---

## Goals

- Standards-based, minimal tooling, maximal longevity
- Token-driven: no hardcoded values, everything via custom properties
- Cascade-aware: `@layer` as the primary architecture mechanism
- Dark-mode ready via `light-dark()` + `color-scheme` (not media query approach)
- Accessible defaults (WCAG 2.2 AA structural baseline)
- Mobile-first without relying on breakpoints

## Layer Architecture

Declared once at the top of `css/main.css`, ordered from least to most specific. **All CSS lives inside a named layer** ŌĆö unlayered CSS automatically trumps all layered CSS, which would silently break the architecture.

```css
@layer reset, tokens, base, layout, utilities, overrides;
```

| Layer | Purpose | Notes |
|---|---|---|
| `reset` | Andy Bell's modern CSS reset | Piccalilli reset + `[hidden]` enforcement + `prefers-reduced-motion` |
| `tokens` | CSS custom properties from Style Dictionary | Two sub-imports: primitive then semantic |
| `base` | Element-level defaults | Typography scale, form normalization, img/media, `:focus-visible` |
| `layout` | Layout primitives | Class-based; Every Layout-inspired; intrinsic |
| `utilities` | Single-purpose helpers | `.visually-hidden`, `.flow`, `.truncate`, `.skip-to-content` |
| `overrides` | Escape hatch (empty by default) | Documented; teams put per-project exceptions here |

**Philosophy (CUBE-influenced):** trust the cascade and inheritance. Layers reduce specificity conflicts without fighting CSS fundamentals. No `!important` except intentionally in `overrides`, and in `reset` for `[hidden]`.

### Reset layer additions (beyond Andy Bell's reset)

```css
/* Enforce hidden attribute ŌĆö some browsers apply it with lower specificity */
[hidden] {
  display: none !important;
}

/* Global motion reduction ŌĆö animations and transitions off by default for users who prefer it */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

### Base layer: `:focus-visible`

In `css/base/focus.css` ŌĆö scoped to `base`, not `reset`, because it uses design token values.

**Technique: double-ring (outline + box-shadow) with forced-colors safety**

```css
:focus-visible {
  /* Primary colored ring ŌĆö preserved in forced-colors mode (color overridden to system color) */
  outline: 3px solid var(--vs-color-border-focus);
  outline-offset: 3px;

  /* Inner "gap" ring ŌĆö creates double-ring visual separation in normal mode */
  /* Uses page bg token so it adapts automatically in dark mode */
  /* Disappears in forced-colors (box-shadow ŌåÆ none) ŌĆö outer outline still shows */
  box-shadow: 0 0 0 6px var(--vs-color-bg-default);
}

/* Remove :focus for pointer users ŌĆö show only for keyboard/programmatic focus */
:focus:not(:focus-visible) {
  outline: none;
  box-shadow: none;
}
```

**Why this approach:**
- `outline` survives Windows High Contrast Mode / `forced-colors: active`; its color is overridden to a system color (e.g. `ButtonText`) ŌĆö still visible
- `box-shadow` is forced to `none` in WHCM ŌĆö the gap ring disappears, leaving one clear ring, which remains WCAG 2.2 SC 2.4.13 compliant
- `box-shadow` uses `--vs-color-bg-default` (semantic bg token) ŌåÆ white gap in light mode, dark gap in dark mode, no hardcoded values
- No `@media (forced-colors: active)` override block needed for this pattern
- Meets WCAG 2.2 SC 2.4.13 (Focus Appearance): 3px outline Ōēź 2px minimum; 3:1 contrast between focused/unfocused states

**Transparent outline alternative (Microsoft pattern):**
If a component uses `box-shadow` as its *only* focus ring (e.g. replacing outline entirely), add a transparent backup that becomes visible in forced-colors:
```css
outline: 2px solid transparent; /* invisible normally; forced-colors overrides to system color */
```
Document this in ARCHITECTURE.md for team reference when building custom components.

Requires `--vs-color-border-focus` in semantic tokens (dedicated focus color, not the same as `color.border.default`; should pass 3:1 contrast on both `--vs-color-bg-default` and `--vs-color-bg-subtle`).

All `@import` statements use `layer()` syntax:

```css
@import './tokens/primitive.css' layer(tokens);
@import './tokens/semantic.css' layer(tokens);
@import './reset/reset.css' layer(reset);
/* etc. */
```

## Token Architecture

**2-tier: Primitive ŌåÆ Semantic**

### Primitive tokens (`tokens/primitive.tokens.json`)

Raw values. No semantics. Named by scale, not intent. Uses W3C DTCG format (`$type`, `$value`).

Categories:
- `color.*` ŌĆö full palette (neutral 0ŌĆō900, plus accent hues in scale steps)
- `space.*` ŌĆö base-4 px scale (1ŌåÆ4px, 2ŌåÆ8px, 4ŌåÆ16px, 8ŌåÆ32px, 16ŌåÆ64pxŌĆ”)
- `font.family.*` ŌĆö system font stacks: sans (`system-ui, sans-serif`), mono, serif
- `font.size.*` ŌĆö modular type scale (sm, base, lg, xl, 2xl, 3xl, 4xl)
- `font.weight.*` ŌĆö regular (400), medium (500), bold (700)
- `font.line-height.*` ŌĆö tight, base, loose
- `radius.*` ŌĆö sm, md, lg, full
- `border.*` ŌĆö widths only

### Semantic tokens (`tokens/semantic.tokens.json`)

Purposeful aliases into primitive tokens. Two values per color token: light + dark.

```json
{
  "color": {
    "text": {
      "default": {
        "$type": "color",
        "$extensions": {
          "vs": {
            "light": "{color.neutral.900}",
            "dark":  "{color.neutral.0}"
          }
        }
      }
    }
  }
}
```

The custom Style Dictionary formatter reads the `vs.light` / `vs.dark` extension and outputs:

```css
:root {
  color-scheme: light dark;
  --vs-color-text-default: light-dark(var(--vs-color-neutral-900), var(--vs-color-neutral-0));
}
```

Non-color semantic tokens (space, typography) have a single value and are output normally.

Semantic color token groups:
- `color.text.*` ŌĆö default, muted, inverse, accent, error, success
- `color.bg.*` ŌĆö default, subtle, inverse
- `color.border.*` ŌĆö default, **focus** (drives `:focus-visible` outline), error
- `color.accent.*` ŌĆö default, hover

The `--vs-color-border-focus` token is the single source for all `:focus-visible` outline color throughout the system.

## Dark Mode via `light-dark()` + `color-scheme`

**Primary approach:** `light-dark()` function (Baseline 2024 Newly Available: Chrome 123+, Firefox 120+, Safari 17.5+) with `color-scheme: light dark` on `:root`.

Why this over media queries:
- Single `:root` block ŌĆö no duplication, no media query wrapper
- Browser UI (form controls, scrollbars) automatically respects scheme
- Manual toggle works for free: `[data-theme="dark"] { color-scheme: dark }` cascades to all `light-dark()` values on descendants

The examples/index.html kitchen sink includes a `<button>` that toggles `color-scheme` on `:root` for demonstration.

**Documented alternative (in ARCHITECTURE.md):** if teams must support older browsers or need server-side rendering of the theme, a `@media (prefers-color-scheme: dark) { :root { ... } }` block is the fallback pattern.

## Custom Property Prefix

**Default prefix: `--vs-`**

Configured in **one place**: `styledictionary.config.js`

```js
const PREFIX = 'vs'; // set to '' for no prefix, or 'my-org' for custom org prefix
```

Style Dictionary's `prefix` option + `outputReferences: true` ensures all cross-references between primitive and semantic tokens update automatically. Changing the prefix is a one-line change + `npm run tokens`.

Examples:
- `--vs-color-neutral-900` (primitive)
- `--vs-color-text-default` (semantic ŌåÆ `light-dark(var(--vs-color-neutral-900), var(--vs-color-neutral-0))`)
- `--vs-space-md` (semantic ŌåÆ `var(--vs-space-4)`)

## Tooling

### Style Dictionary v4 (token build)

- Input: `tokens/*.tokens.json` (DTCG format)
- Output: `css/tokens/primitive.css` + `css/tokens/semantic.css`
- Custom format in `styledictionary.config.js`: reads `$extensions.vs.light/dark` and outputs `light-dark()` values for color tokens
- Script: `npm run tokens`

### LightningCSS (CSS build)

Replaces PostCSS entirely. Handles:
- `@import` bundling (the primary need)
- CSS nesting passthrough (native nesting is Baseline 2024)
- Vendor prefixes for remaining targets
- Minification for dist

`build.mjs` script:
- Dev: unbundled, source map, no minify ŌåÆ writes to `dist/css/main.css` + `main.css.map`
- Prod/dist: bundled, minified, no source map ŌåÆ `dist/css/main.css`

Source maps are NOT included in the npm-published package (`files` in `package.json` excludes `*.map`).

Browser targets: `browserslist` (`> 0.5%, last 2 versions, not dead`) ŌåÆ `browserslist-to-targets` helper for LightningCSS.

### Biome (formatting + basic lint)

- Formatter for JS, JSON, CSS: tabs, double quotes
- CSS linter: basic rules enabled
- Does NOT cover architecture-critical CSS rules (Stylelint handles those)

### Stylelint (minimal architecture linting)

Extends `stylelint-config-standard`. Overrides kept to ~6 rules:

1. `color-named: error` ŌĆö no raw named colors; use tokens
2. No missing `var()` wrapper on custom properties
3. `no-descending-specificity` ŌĆö specificity ordering enforcement
4. `declaration-property-value-no-unknown` ŌĆö catches typos in values
5. Modern color notation enforcement (`oklch`/`color()` over legacy `rgb()`)
6. BEM selector pattern ŌĆö regex enforcing `.block__element--modifier`

## Layout Primitives (7 classes, 1 modifier)

All in `css/layout/`. Class names are working names; renaming is a single-file change.

| File | Class | Mechanism |
|---|---|---|
| `stack.css` | `.stack` | Flex column + gap via `--vs-space-md` |
| `cluster.css` | `.cluster` | Flex wrap + gap; adapts to available space |
| `center.css` | `.center` | Max-width container + `margin-inline: auto` |
| `sidebar.css` | `.sidebar` | Flex row; sidebar collapses below threshold |
| `grid.css` | `.grid` | CSS Grid `auto-fill` + `minmax`; no media queries |
| `switcher.css` | `.switcher` | Flex; switches axis when total width < threshold |
| `prose.css` | `.prose` + `.breakout` | Reading context + full-bleed child escape |

All use semantic space tokens for gap/padding. No hardcoded values. No media queries ŌĆö intrinsic/algorithmic layout only.

### `.prose` + `.breakout` detail

`.prose` provides a reading context: constrained width (~65ch via `--vs-prose-width` custom property), comfortable line height for body copy. It becomes a CSS Grid container with named column lines so `.breakout` children can escape the constraint:

```css
.prose {
  --vs-prose-width: 65ch;
  display: grid;
  grid-template-columns:
    [full-start] minmax(0, 1fr)
    [prose-start] min(var(--vs-prose-width), 100%)
    [prose-end] minmax(0, 1fr)
    [full-end];
}

.prose > * {
  grid-column: prose;
}

.breakout {
  grid-column: full;
}
```

`.breakout` is contextual ŌĆö it only works as a direct child of `.prose`. Documented in ARCHITECTURE.md. The `--vs-prose-width` custom property is overridable per-instance.

## Utilities

| File | Class | Purpose |
|---|---|---|
| `visually-hidden.css` | `.visually-hidden` | Accessible hide (screen reader visible, layout hidden) |
| `flow.css` | `.flow` | Margin-top vertical rhythm for prose/body text |
| `truncate.css` | `.truncate` | Text truncation with ellipsis; single or multi-line |
| `skip-to-content.css` | `.skip-to-content` | Keyboard nav skip link; visible on focus |

### `.truncate` detail

Uses `-webkit-line-clamp` (Baseline 2020, widely available) with a CSS custom property for line count. Single-line by default, multi-line via `--truncate-lines`:

```css
.truncate {
  display: -webkit-box;
  -webkit-box-orient: vertical;
  -webkit-line-clamp: var(--truncate-lines, 1);
  overflow: hidden;
}
```

Usage:
```html
<!-- Single line (default) -->
<p class="truncate">Long textŌĆ”</p>

<!-- Three lines -->
<p class="truncate" style="--truncate-lines: 3">Long textŌĆ”</p>
```

`-webkit-line-clamp: 1` behaves identically to the classic `white-space: nowrap; overflow: hidden; text-overflow: ellipsis` pattern in all modern browsers, so this single implementation covers both cases without the `white-space` conflict.

## Assumptions

1. All CSS is global (no scoped/component styles)
2. LightningCSS Node API (not a Vite/Webpack plugin) ŌĆö zero framework coupling
3. Style Dictionary v4 (ESM-first, DTCG native support)
4. Biome v2+ (CSS formatter/linter support included)
5. `light-dark()` is acceptable as Baseline 2024 Newly Available (not Widely Available yet)
6. Layout class names (`.stack`, `.cluster`, etc.) are finalized during implementation
7. No SCSS ŌĆö native CSS only; nesting is Baseline 2024

## Excluded (with rationale)

| Thing | Why excluded |
|---|---|
| `dist/css/layers/` | Individual layer files aren't standalone (depend on token custom properties); no useful npm artifact; template users get source files via giget |
| Tailwind | Explicitly out of scope |
| PostCSS | Replaced by LightningCSS; simpler, faster, fewer deps |
| SCSS/Sass | Native CSS nesting is Baseline 2024 |
| Component-level tokens (3rd tier) | Out of scope; teams add their own component layer |
| Full Stylelint config | Too heavy for a boilerplate; architecture rules only |

| Media query for dark mode | Replaced by `light-dark()` + `color-scheme`; documented as fallback |
| Utopia fluid type/space | Too opinionated for a generic boilerplate; static scale in tokens instead |
